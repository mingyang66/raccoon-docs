##### springboot将Filter注册到servlet容器后执行顺序两种定义方案

> 最近闲来无事翻看之前写的过滤器代码，发现通过FilterRegistrationBean将过滤器注入到spring容器中时，过滤器中定义的order顺序是无效的，便通过一步步的调试源代码发现了其中的实现原理。

##### 一、定义过滤器（指定优先级顺序）

```java
/**
 * 拦截所有请求过滤器，并将请求类型是HttpServletRequest类型的请求替换为自定义{@link ContentCachingRequestWrapper}
 *
 * @author Emily
 * @since 2020/8/19
 */
public class OrderedContentCachingRequestFilter extends OncePerRequestFilter implements OrderedFilter {

    private static final Logger LOG = LoggerFactory.getLogger(OrderedContentCachingRequestFilter.class);
    /**
     * 顺序必须在{@link OrderedRequestContextFilter}之后，否则拿不到HttpServletRequest对象
     */
    private int order = -104;

    @Override
    protected void doFilterInternal(@Nonnull HttpServletRequest request, @Nonnull HttpServletResponse response, @Nonnull FilterChain filterChain) throws ServletException, IOException {
        //创建ContentCachingRequestWrapper对象用于缓存请求体
        ContentCachingRequestWrapper requestWrapper = new ContentCachingRequestWrapper(request);
        if (LOG.isDebugEnabled()) {
            LOG.debug("请求接口缓存拦截器：START============>>{}", FilterUtil.getRequestPath(request));
        }
        //标记阶段标识
        LocalContextHolder.current().setTracingPhase(TracingPhase.PARAMETER);
        //继续执行过滤器链，并传递包装后的请求对象
        filterChain.doFilter(requestWrapper, response);
        //移除线程上下文数据
        LocalContextHolder.unbind(true);
        if (LOG.isDebugEnabled()) {
            LOG.debug("请求接口缓存拦截器：END<<============{}", FilterUtil.getRequestPath(request));
        }
    }

    @Override
    public int getOrder() {
        return order;
    }

    public void setOrder(int order) {
        this.order = order;
    }
}
```

##### 二、将过滤器注入到spring容器中

- 指定优先级顺序，方案一

```java
    /**
     * 注册HTTP请求拦截器注册BEAN
     * 1. 过滤器执行优先级必须在{@link RequestContextFilter}({@link OrderedRequestContextFilter} -105) 之后，否则拿不到HttpServletRequest对象；
     *
     * @return 过滤器注册对象
     */
    @Bean
    @Role(BeanDefinition.ROLE_INFRASTRUCTURE)
    public FilterRegistrationBean<OrderedContentCachingRequestFilter> filterRegistrationBean() {
        FilterRegistrationBean<OrderedContentCachingRequestFilter> filterRegistrationBean = new FilterRegistrationBean<>();
        filterRegistrationBean.setName("contentCachingWrapper");
        filterRegistrationBean.setFilter(new OrderedContentCachingRequestFilter());
        filterRegistrationBean.setUrlPatterns(List.of("/*"));
        filterRegistrationBean.setOrder(-104);
        return filterRegistrationBean;
    }
```

> 此时会发现我在OrderedContentCachingRequestFilter过滤器和FilterRegistrationBean中都指定了优先级顺序，如果修改过滤器的优先级顺序，不会影响到过滤器实际的执行顺序，真实起作用的是FilterRegistrationBean指定的优先级顺序；这时就困惑了，那过滤器中指定优先级顺序有啥用。

- 指定优先级顺序，方案二

```java
    @Bean
    @ConditionalOnMissingBean
    @Role(BeanDefinition.ROLE_INFRASTRUCTURE)
    public OrderedContentCachingRequestFilter contentCachingRequestFilter() {
        return new OrderedContentCachingRequestFilter();
    }
```

> 此时会将过滤器注入到spring容器中，修改过滤器的优先级顺序，则会根据不同的值按照不同的顺序执行。

- 结论

通过FilterRegistrationBean注册过滤器到容器中去时，过滤器无需指定优先级顺序；直接将过滤器注册到容器中去时需要指定优先级顺序；

##### 三、注册过滤器到容器中的两种方案源码分析

- ServletContextInitializerBeans类构造函数

```java
    @SafeVarargs

        this.addServletContextInitializerBeans(beanFactory);
       // 通过适配器模式将过滤器转换为FilterRegistrationBean注册到集合中
        this.addAdaptableBeans(beanFactory);
```
