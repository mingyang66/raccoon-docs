###### 一、Maven依赖配置需引入spring-boot-starter-mail模块

```xml
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-mail</artifactId>
        </dependency>
```

###### 二、邮箱配置（以QQ邮箱为例）

在application.properties中配置SMTP参数，注意密码使用授权码而非登录密码：

```properties
#SMTPTransport读取属性配置类
#smtp地址
spring.mail.host=smtp.qq.com
#端口号，465【SSL】、587【TLS】
spring.mail.port=465
#邮箱地址
spring.mail.username=xxx@foxmail.com
#授权码
spring.mail.password=xxxxxxx
#协议类型，默认：smtp, smtps代表开启SSL
spring.mail.protocol=smtp
#编码方式，默认：UTF-8
spring.mail.default-encoding=UTF-8
#启用SMTP身份认证机制【设置了spring.mail.username、spring.mail.password效果一样】
spring.mail.properties.mail.smtp.auth=false
#SSL加密，默认：false【设置spring.mail.protocol=smtps效果一样】
spring.mail.properties.mail.smtp.ssl.enable=true
#尝试升级TLS加密，升级失败会退回明文，默认：false
spring.mail.properties.mail.smtp.starttls.enable=false
#强制要求必须成功建立SSL连接，否则立即终止，默认：false
spring.mail.properties.mail.smtp.starttls.required=false
#建立连接超时，单位：毫秒，默认：-1
spring.mail.properties.mail.smtp.connectiontimeout=5000
#smtp协议读取响应超时，单位：毫秒，默认：-1
spring.mail.properties.mail.smtp.timeout=5000
#smtp协议写入数据超时，单位：毫秒，默认：-1
spring.mail.properties.mail.smtp.writetimeout=5000

#开启调试日志，定位具体认证失败的原因
spring.mail.properties.mail.debug=true
#启用后会输出SMTP认证过程的详细日志（包括BASE64编码的认证信息）,默认：false
spring.mail.properties.mail.debug.auth=true
#调试模式下显式指定的认证用户名（通常无需单独配置）,默认：true
spring.mail.properties.mail.debug.auth.username=true
#调试模式下显式指定的认证密码（存在安全风险，慎用）,默认: false
spring.mail.properties.mail.debug.auth.password=true
#是否开启连接有效性验证，默认：false
spring.mail.test-connection=true
```

###### 三、简单文本邮件发送

```java
@RestController
public class EmailController {
    private final JavaMailSender mailSender;

    public EmailController(JavaMailSender mailSender) {
        this.mailSender = mailSender;
    }

    @GetMapping("api/v1/email/sendSimple")
    public void sendSimpleMail(String from, String to, String subject, String content) {
        SimpleMailMessage message = new SimpleMailMessage();
        message.setFrom(from);
        message.setTo(to);
        message.setSubject(subject);
        message.setText(content);
        mailSender.send(message);
    }
  }
```

###### 四、HTML格式邮件发送

```java
    @GetMapping("api/v1/email/sendHtml")
    public void sendHtmlMail(String from, String to, String subject, String content)
            throws MessagingException {
        MimeMessage message = mailSender.createMimeMessage();
        MimeMessageHelper helper = new MimeMessageHelper(message, true);
        //发件人
        helper.setFrom(from);
        //收件人
        helper.setTo(to);
        //抄送收件人
        helper.setCc("CC@QQ.COM");
        //密件抄送收件人（其它收件人无法看到BCC中的邮件列表）
        helper.setBcc("xx@eastmoney.com");
        //回复收件人
        helper.setReplyTo("dd@qq.com");
        helper.setSubject(subject);
        helper.setText(content, true); //true标识html格式
        helper.addInline("logo", new ClassPathResource("image/12.png"));
        helper.addAttachment("document.pdf", new ClassPathResource("image/file.pdf"));

        mailSender.send(message);
    }
```

###### 八、启用SMTP的身份验证机制

```properties
spring.mail.properties.mail.smtp.auth=true
```

> 此配置是启用SMTP的身份验证机制，确保邮件发送请求需要通过账号密码（授权码）认证才能执行。

此配置设置为true通如下配置效果一样：

```properties
spring.mail.username=xx@foxmail.com
spring.mail.password=abcdxdfzddd
```

源码如下（org.eclipse.angus.mail.smtp.SMTPTransport#protocolConnect）：

            }

            if (port == -1) {

            }

            boolean connected = false;

                    if (this.serverSocket instanceof SSLSocket) {
                        this.logger.fine("STARTTLS requested but already using SSL");

                        this.logger.fine("STARTTLS required but not supported");

                    }

                }

                if ((useAuth || user != null && password != null) && (this.supportsExtension("AUTH") || this.supportsExtension("AUTH=LOGIN"))) {
