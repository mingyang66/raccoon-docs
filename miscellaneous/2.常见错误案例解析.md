##### 一、对象类型为null，自动拆箱空指针异常

```java
public class Test {
    public static void main(String[] args) {
        System.out.println(compute(null));
    }

    public static long compute(Long a) {
        long b = 9;
        return a + b;
    }
}
```

运行上述案例会报如下错误：

```sh
Exception in thread "main" java.lang.NullPointerException
	at com.eastmoney.emis.trade.account.Test.compute(Test.java:13)
	at com.eastmoney.emis.trade.account.Test.main(Test.java:9)
```

> 解析：compute方法中，当传入的参数a为null时，执行return a + b;触发了自动拆箱操作。当Long类型a为null时，在表达式a + b中，a需要被自动拆箱为基本数据类型long才能与b相加。这个拆箱过程实际上调用了a.longValue()方法。由于a是null，调用longValue()方法自然就抛出java.lang.NullPointerException。

其它案例，如赋值操作也会发生空指针异常，如下：

```java
public class Test {
    public static void main(String[] args) {
        System.out.println(compute(null));
    }

    public static long compute(Long a) {
        long b = a;
        //todo 逻辑
        return b;
    }
}
```

##### 二、Collectors.toMap()方法

Collectors.toMap()是Java Stream API中用于将流元素收集到Map中的强大工具，它支持灵活的键值映射和冲突处理机制。

toMap()方法主要有三种重载形式：

```java
// 两参数形式 - 简单映射，遇到重复键会抛出异常
toMap(Function keyMapper, Function valueMapper)

// 三参数形式 - 包含合并函数处理重复键
toMap(Function keyMapper, Function valueMapper, BinaryOperator mergeFunction)

// 四参数形式 - 额外指定Map实现
toMap(Function keyMapper, Function valueMapper, BinaryOperator mergeFunction, Supplier mapSupplier)

```

- 键映射器（keyMapper）：定义如何从流元素中提取Map的键。
- 值映射器（valueMapper）：定义如何从流元素中提取Map的值。
- 冲突处理器（mergeFunction）：当多个元素映射到同一个键时，默认会抛出IllegalStateException异常，通过三参数形式可以定义合并策略：

```java
// 保留旧值
(oldValue, newValue) -> oldValue

// 使用新值
(oldValue, newValue) -> newValue

// 合并值（如字符串拼接）
(oldValue, newValue) -> oldValue + ", " + newValue

// 根据业务规则选择（如年龄较大的对象）
(a, b) -> a.getAge() > b.getAge() ? a : b

```

- 指定Map实现类（mapSupplier）：第四个参数允许指定具体的Map实现类，如LinkedHashMap、TreeMap等。

  

  两参数形式（当出现重复键的时候会抛出IllegalStateException异常）：

```java
// 示例：学生ID到姓名的映射
List<Student> students = Arrays.asList(
    new Student(1, "张三"),
    new Student(2, "李四"),
    new Student(3, "王五")
);

Map<Integer, String> studentMap = students.stream()
    .collect(Collectors.toMap(Student::getId, Student::getName));
// 结果：{1=张三, 2=李四, 3=王五}

```

三参数形式（包含合并策略）：

```java
// 示例：处理重复产品ID
List<Product> products = Arrays.asList(
    new Product(1L, "苹果"),
    new Product(1L, "香蕉"), // 重复键
    new Product(2L, "橙子")
);

// 保留旧值策略
Map<Long, String> productMap1 = products.stream()
    .collect(Collectors.toMap(
        Product::getId,
        Product::getName,
        (oldValue, newValue) -> oldValue
    ));

// 使用新值策略  
Map<Long, String> productMap2 = products.stream()
    .collect(Collectors.toMap(
        Product::getId,
        Product::getName,
        (oldValue, newValue) -> newValue
    ));

// 复杂合并：选择销量最高的专辑
Map<Artist, Album> topHits = albums.stream()
    .collect(Collectors.toMap(
        Album::getArtist,
        a -> a,
        BinaryOperator.maxBy(Comparator.comparing(Album::getSales))
    ));

```

四参数形式（指定Map实现）：

```java
// 示例：保持插入顺序的LinkedHashMap
Map<Integer, Student> orderedMap = students.stream()
    .collect(Collectors.toMap(
        Student::getId,
        student -> student,
        (oldValue, newValue) -> newValue, // 合并函数
        LinkedHashMap::new // Map工厂
    ));

```

##### 三、Collectors.groupingBy()方法

Collectors.groupingBy()是Java8 Stream API中用于数据分组的重要方法，它类似于SQL中的GROUP BY字句。

groupingBy()方法有如下三种重载方法：

```java
groupingBy(Function<? super T, ? extends K> classifier)
groupingBy(Function<? super T, ? extends K> classifier,Collector<? super T, A, D> downstream) 
groupingBy(Function<? super T, ? extends K> classifier,Supplier<M> mapFactory,Collector<? super T, A, D> downstream)
```

- classifier（分类器函数）：从输入元素中提取分组键；
- downstream（下游收集器）：定义分组值的聚合方式，如：计数、求平均值、集合等；
- mapFactory（Map工厂）：提供返回的Map容器类型；

```java
LinkedHashMap<String, List<ProcessDetails>> resultSort = result.stream()
    .sorted(Comparator.comparing(ProcessDetails::getBeginTime))
    .collect(Collectors.groupingBy(
        ProcessDetails::getGroupBy,        // 分类器
        LinkedHashMap::new,               // Map工厂
        Collectors.toList()               // 下游收集器
    ));

```

案例一：多级分组

```java
Map<外层键, Map<内层键, 列表类型>> result = list.stream()
    .collect(Collectors.groupingBy(
        外层分类函数,
        Collectors.groupingBy(内层分类函数)
    ));

```

在多级分组的每一级都可以结合下游收集器进行统计操作:

```java
// 先按部门分组，再按城市统计平均年龄
Map<String, Map<String, Double>> deptCityAvgAge = employees.stream()
    .collect(Collectors.groupingBy(
        Employee::getDepartment,
        Collectors.groupingBy(
            Employee::getCity,
            Collectors.averagingInt(Employee::getAge)
        )
    ));

```

分组后转换为其它集合类型，在多级分组的最后一级，可以将结果转换为Set等其它集合类型：

```java
Map<String, Map<String, Set<String>>> result = list.stream()
    .collect(Collectors.groupingBy(
        Student::getBatchNumber,
        Collectors.groupingBy(
            Student::getType,
            Collectors.mapping(Student::getStatus, Collectors.toSet())
        )
    ));

```

案例二：

```java
list.stream().collect(Collectors.groupingBy(A::getGroupDate, TreeMap::new, toList())).descendingMap();
```

键参数解析：

- A::getGroupDate：分类器函数，按照getGroupDate方法返回值分组；
- TreeMap::new：Map工厂，指定使用TreeMap作为结果容器；
- toList()：下游收集器，将分组元素收集到List中；

TreeMap排序机制：

当使用TreeMap::new作为mapFactory参数时，分组结果会按照键的自然顺序进行排序，TreeMap底层使用红黑树数据结构，能够自动维护键的有序性。

descendingMap()方法的作用：

descendingMap()是TreeMap特有的方法，它会返回当前映射的逆序视图，例如：如果原始键的顺序是[1,2,3]，调用descendingMap后会变成[3,2,1]。

与其它Map对比：

- HashMap：默认实现，不保证任何顺序；
- LinkedHashMap：保持元素插入顺序；
- TreeMap：按键的自然顺序排序；

##### 四、@Scheduled标注方法抛出异常是否会终止

在Spring框架中使用@Scheduled注解的定时任务中，如果方法抛出异常，该定时任务不会终止，而是会继续按照设定的时间间隔执行。

```java
    @Scheduled(cron = "0/5 * * * * ?")
    public void task(){
        System.out.println("---start-----");
        throw new IllegalArgumentException("dfs");
        //System.out.println("---end-----");
    }
```

> 上述案例即使发生异常，也会每隔5秒钟执行一次，Spring调度机制会捕获这些异常，记录错误日志，但不会终端后续的调度执行。这就意味着即使当前执行失败，5s后依旧会执行。
