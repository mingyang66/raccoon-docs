PKCS#8是一种标准的私钥存储格式语法，在Java开发和现代加密体系中，它是最常用的私钥格式。

##### 一、核心特征

- 通用性：它是语言无关的标准格式，Java(java.security.spec.PKCS8EncodedKeySpec)、OpenSSL、.NET、Node.js等都原生支持。

- 算法无关：它可以封装RSA、EC(椭圆曲线)、DSA等各种算法的私钥，而不仅仅是RSA。

- 头部标记，未加密：
  
      -----BEGIN PRIVATE KEY-----

- 头部标记，已加密：
  
      -----BEGIN ENCRYPTED PRIVATE KEY-----

- Java对应类：java.security.spec.PKCS8EncodedKeySpec

##### 二、PKCS#8 vs 传统RSA格式（SSLeay）

这是最容易混淆的地方。OpenSSL默认生成的往往是传统格式，但是最新版本生成的都是PKCS#8格式了。

| 特性         | PKCS#8 (现代标准)                                                          | 传统 RSA 格式 (SSLeay / PKCS#1)                                                                                                            |
| ---------- | ---------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| PEM 头标记    | `-----BEGIN PRIVATE KEY-----`<br>*(加密时为 `ENCRYPTED PRIVATE KEY`)*      | `-----BEGIN RSA PRIVATE KEY-----`                                                                                                      |
| 结构内容       | 通用容器：包含算法 OID 标识符 + 私钥数据。<br>*(可封装 RSA, EC, DSA 等任意算法)*                | 算法特定：仅包含 RSA 算法特有的参数 (n, e, d, p, q 等)。<br>*(无法封装非 RSA 密钥)*                                                                            |
| Java 支持    | 首选/原生：<br>`PKCS8EncodedKeySpec` 直接支持。<br>*(JDK 所有版本均完美支持)*             | 有限支持：<br>1. `RSAPrivateCrtKeySpec` 可解析，但代码繁琐。<br>2. 较新 JDK (如 15+) 或特定 Provider 可能直接拒绝加载此格式，报错 `InvalidKeyException`。<br>3. 强烈建议转换后使用。 |
| OpenSSL 生成 | 非默认：<br>需显式转换：`openssl pkcs8 -topk8 ...`<br>*(新版 `genpkey` 命令默认生成此格式)* | 默认行为：<br>旧命令 `openssl genrsa` 默认生成此格式。<br>*(这是造成 Java 兼容性问题最常见的根源)*                                                                    |
| 适用场景       | 现代开发标准：<br>微服务、Spring Boot、云原生、跨语言交互 (Go, Node, Python, Java)。         | 遗留系统：<br>旧版 OpenSSL 脚本、仅针对 RSA 的老旧 C/C++ 程序、部分硬件安全模块 (HSM) 导出格式。                                                                       |

如何将一个传统格式的转换为PKCS#8格式的：

    # -nocrypt 表示不设置密码，生成纯文本私钥
    openssl pkcs8 -topk8 -inform PEM -outform PEM -nocrypt -in rsa_private_key.pem -out pkcs8_private_key.pem

较新版本的OpenSSL可以直接生成PKCS#8格式的：

    # 注意：genpkey 是比 genrsa 更通用的命令
    openssl genpkey -algorithm RSA -out pkcs8_private_key.pem -pkeyopt rsa_keygen_bits:2048
