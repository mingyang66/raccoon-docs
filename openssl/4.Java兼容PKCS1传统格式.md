##### 一、兼容PKCS#1格式

        /**
         * 从字符串加载私钥 (自动清洗 PEM 格式)
         */
        private static RSAPrivateKey loadPrivateKeyFromString(String keyStr) throws Exception {
            byte[] keyBytes = Base64.getDecoder().decode(keyStr);
            Security.addProvider(new BouncyCastleProvider());
            PKCS8EncodedKeySpec spec = new PKCS8EncodedKeySpec(keyBytes);
            KeyFactory keyFactory = KeyFactory.getInstance(ALGORITHM);
            return (RSAPrivateKey) keyFactory.generatePrivate(spec);
        }

上述代码新增了一条Security.addProvider(new BouncyCastleProvider())代码是为了兼容传统的PKCS#1格式的私钥。

需要引入如下三方包：

    <!-- Source: https://mvnrepository.com/artifact/org.bouncycastle/bcprov-jdk18on -->
    <dependency>
        <groupId>org.bouncycastle</groupId>
        <artifactId>bcprov-jdk18on</artifactId>
        <version>1.83</version>
        <scope>compile</scope>
    </dependency>

bcprov-jdk18on 是 Bouncy Castle 专门针对 JDK 18 及更高版本 优化的核心加密库。
如果你正在使用 Java 18、21、23 或更新的 LTS/短期版本，强烈建议使用这个版本，而不是旧的 jdk15on。虽然旧版本通常也能运行，但新版本修复了 JDK 内部 API 变更导致的兼容性问题，并提供了更好的性能和安全支持。

##### 二、那些场景需要引入此包

| 场景      | 私钥格式特征                                 | 是否需要 Bouncy Castle? | 推荐方案                           |
| ------- | -------------------------------------- | ------------------- | ------------------------------ |
| 标准未加密   | `BEGIN PRIVATE KEY`<br>(无密码)           | ❌ 不需要               | JDK 原生 (`PKCS8EncodedKeySpec`) |
| 加密保护    | `BEGIN ENCRYPTED PRIVATE KEY`<br>(有密码) | ✅ 需要                | BC (`PEMParser` 解密)            |
| 传统格式    | `BEGIN RSA PRIVATE KEY`<br>(PKCS#1)    | ✅ 需要                | BC (解析转换) 或 OpenSSL 预转换        |
| 国密/特殊算法 | SM2, 特殊 OID                            | ✅ 需要                | BC (提供算法实现)                    |
